# Death AI Integration Guide
**Version:** 1.0  
**Last Updated:** 2025-11-06  
**Component:** DeathAI Cheating Detection & Adaptive Difficulty

---

## Overview

This guide shows how to integrate the new DeathAI features:
1. **Cheating Detection** - Detects save scumming, infinite items, invulnerability, room skipping
2. **Adaptive Difficulty** - Automatically adjusts challenge based on player performance

---

## Integration Points

### 1. Save/Load System Integration

**When player loads a save, call:**
```python
# In GameLogic.load_game() or wherever saves are loaded
def load_game(self, save_file):
    # ... existing load logic ...
    
    # Notify Death AI about save load
    if self.death_ai:
        self.death_ai.on_save_loaded()
    
    # ... rest of load logic ...
```

**What it does:**
- Tracks reload timestamps
- Detects if player reloads >5 times in 10 minutes
- Triggers "save scumming" punishment (harder hazards, +30% fear, Death taunts)

---

### 2. Item System Integration

**When player takes an item, call:**
```python
# In GameLogic.handle_take() or item pickup logic
def handle_take(self, item_key):
    current_room = self.player.get('location', '')
    
    # ... existing take logic ...
    
    # Notify Death AI about item taken
    if self.death_ai and current_room:
        self.death_ai.on_item_taken(item_key, current_room)
    
    # ... rest of take logic ...
```

**What it does:**
- Tracks item_id + room_id combinations
- Detects if same item taken >3 times from same room
- Triggers "infinite items" punishment (item removed, Death warns)

---

### 3. Damage System Integration

**When player takes damage, call:**
```python
# In GameLogic.apply_damage() or wherever HP is reduced
def apply_damage(self, damage_amount):
    old_hp = self.player.get('hp', 100)
    
    # Apply damage
    self.player['hp'] = max(0, old_hp - damage_amount)
    
    # Notify Death AI about damage taken
    if self.death_ai and damage_amount > 0:
        self.death_ai.on_damage_taken(damage_amount)
    
    # ... rest of damage logic ...
```

**What it does:**
- Tracks damage events and resulting health values
- Detects if player takes damage but health doesn't decrease (invulnerability)
- Triggers "invulnerability" punishment (instant death, max fear, Death is furious)

### 4. Room Navigation Integration

**When player enters a room, call:**
```python
# In GameLogic.handle_move() or room transition logic
def handle_move(self, direction):
    target_room_id = self.current_room_data['exits'].get(direction)
    
    if not target_room_id:
        self.add_message("You can't go that way.")
        return
    
    # Check if room is locked
    was_locked = self._is_room_locked(target_room_id)
    used_key = False
    
    if was_locked:
        # Try to unlock with key
        required_key = self._get_required_key(target_room_id)
        if required_key in self.player.get('inventory', []):
            used_key = True
            self.add_message(f"You unlock the door with {required_key}.")
        else:
            self.add_message("The door is locked.")
            return
    
    # Move to room
    self.player['location'] = target_room_id
    
    # Notify Death AI about room entry
    if self.death_ai:
        self.death_ai.on_room_entered(target_room_id, was_locked, used_key)
    
    # ... rest of move logic ...
```

**What it does:**
- Tracks room entry with lock status
- Detects if player enters locked room without using key (room skipping exploit)
- Triggers "room skipping" punishment (hazard spawned, +40% fear, Death warns)

### 5. Performance Tracking Integration

**Track player performance for adaptive difficulty:**

**After hazard cleared:**
```python
# In HazardEngine or GameLogic after successful hazard clear
def on_hazard_cleared(self, hazard_id):
    # ... existing clear logic ...
    
    # Update Death AI performance tracking
    if self.death_ai:
        self.death_ai.update_performance_tracking('hazard_cleared', success=True)
```

**After QTE completion:**
```python
# In QTE_Engine after QTE completes
def on_qte_complete(self, errors):
    success = (errors == 0)
    
    # ... existing QTE logic ...
    
    # Update Death AI performance tracking
    if hasattr(self.game_logic, 'death_ai') and self.game_logic.death_ai:
        if errors == 0:
            self.game_logic.death_ai.update_performance_tracking('qte_perfect', success=True)
        else:
            self.game_logic.death_ai.update_performance_tracking('qte', success=(errors < 3))
```

**After player death:**
```python
# In GameLogic when player dies
def on_player_death(self, cause):
    # ... existing death logic ...
    
    # Update Death AI performance tracking
    if self.death_ai:
        self.death_ai.update_performance_tracking('death', success=False)
    
    # ... trigger death screen, respawn, etc ...
```

**After NPC saved:**
```python
# In GameLogic when NPC fate becomes "saved"
def on_npc_saved(self, npc_id):
    # ... existing NPC logic ...
    
    # Update Death AI performance tracking
    if self.death_ai:
        self.death_ai.update_performance_tracking('npc_saved', success=True)
```

**What it does:**
- Tracks recent player actions (last 10 stored)
- Calculates success rate and performance metrics
- Automatically adjusts difficulty if player is doing too well or struggling
- Death speaks when difficulty changes

## 6. Difficulty Modifier Usage

**When calculating QTE time:**
```python
# In QTE_Engine or HazardEngine when setting up QTE
def start_qte(self, base_time):
    # Get difficulty-modified time
    if hasattr(self.game_logic, 'death_ai') and self.game_logic.death_ai:
        modified_time = self.game_logic.death_ai.get_difficulty_modified_value(
            base_time, 
            'qte_time'
        )
    else:
        modified_time = base_time
    
    # Use modified_time for QTE
    self.qte_time_limit = modified_time
```

**When applying damage:**
```python
# In GameLogic or HazardEngine when calculating damage
def calculate_damage(self, base_damage):
    # Get difficulty-modified damage
    if self.death_ai:
        modified_damage = self.death_ai.get_difficulty_modified_value(
            base_damage,
            'damage'
        )
    else:
        modified_damage = base_damage
    
    return int(modified_damage)
```

**When adding fear:**

## Performance Considerations

**Memory Usage:**
- `reload_timestamps`: Max 20 entries (deque with maxlen=20)
- `damage_events`: Max 10 entries (trimmed after each addition)
- `recent_performance_window`: Max 10 entries (deque with maxlen=10)
- `item_takes`: Unlimited dict, but only tracks taken items (reasonable size)

**CPU Usage:**
- All detection methods are O(1) or O(n) where n ≤ 20
- No expensive operations or background threads
- Safe to call on every action

## Known Issues & Edge Cases

### Issue 1: False Positive Save Scumming
**Scenario:** Player legitimately trying different strategies  
**Solution:** Threshold is 5 reloads in 10 minutes, which is generous  
**Workaround:** Adjust threshold higher if needed (e.g., 8 reloads)

### Issue 2: Items That Respawn
**Scenario:** Some items might be designed to respawn  
**Solution:** Mark respawnable items in JSON with `"respawns": true`  
**Code:**
```python
def on_item_taken(self, item_id: str, room_id: str):
    # Check if item is meant to respawn
    item_data = self.game_logic.get_item_data(item_id)
    if item_data and item_data.get('respawns', False):
        return  # Don't track respawning items
    
    # ... rest of method ...
```

### Issue 3: Legitimate Room Access Without Keys
**Scenario:** Some rooms might have alternate unlocking methods (puzzles, NPCs)  
**Solution:** Track all legitimate unlock methods  
**Code:**
```python
# When room unlocked by puzzle/NPC
self.death_ai.cheating_metadata['room_unlocks'][room_id].append(True)

# Then on_room_entered won't trigger punishment
```
