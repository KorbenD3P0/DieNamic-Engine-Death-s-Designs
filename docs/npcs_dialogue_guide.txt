## Overview

NPCs (Non-Player Characters) populate the game world, providing:
- **Narrative depth**: Backstory, motivation, personality
- **Player choices**: Save, ignore, or inadvertently doom NPCs
- **Moral weight**: Decisions have consequences
- **Information**: Hints, clues, warnings about hazards
- **Companionship**: Reduce isolation in horror setting

### Design Philosophy

**NPCs should:**
- Have clear personality and motivation
- React to player actions and world events
- Provide meaningful choices (save vs. self-preservation)
- Face consequences tied to player decisions
- Enhance narrative (not just quest dispensers)

**NPCs should NOT:**
- Be invincible (Death claims everyone eventually)
- Solve puzzles for player (hints only)
- Be numerous (quality over quantity)
- Lack personality (generic dialogue)

### NPC Types

| Type | Purpose | Examples | Survives? |
|------|---------|----------|-----------|
| **Visionary** | Sees death coming, guides player | Corbin Wainwright | Sometimes |
| **Survivor** | Fellow trapped victim | Fairgoers, patients | Rarely |
| **Authority** | Doctor, security, staff | Dr. Ellis, Guard Mason | Rarely |
| **Bystander** | Background character | Waiting room patient | Never |
| **Death** | Antagonist, narrator | Death personified | Always |

## NPC Anatomy

### File Location
`data/npcs.json`

### Minimal Required Schema
```json
{
  "npc_name": {
    "name": "NPC Name",
    "description": "Brief description",
    "dialogue": {
      "first_encounter": {
        "text": "What the NPC says on first meeting"
      }
    }
  }
}
```

### Complete Schema
```json
{
  "npc_key": {
    "name": "Display Name",
    "description": "Physical description and demeanor",
    "initial_state": "conscious",
    "current_state": "conscious",
    "location": "Room Name",
    "is_visionary": false,
    "personality_traits": ["cautious", "analytical"],
    "dialogue": {
      "first_encounter": {
        "text": "First meeting dialogue",
        "gives_item": "item_key",
        "unlocks_achievement": "achievement_id",
        "changes_npc_state": "state_key"
      },
      "repeat_encounter": {
        "text": "Subsequent meetings dialogue"
      },
      "low_health": {
        "text": "Dialogue when NPC is injured",
        "condition": "health < 30"
      },
      "hazard_active": {
        "text": "Dialogue when hazard nearby",
        "condition": "hazard_active:ventilator"
      },
      "after_quest": {
        "text": "Dialogue after specific event",
        "condition": "quest_completed:save_corbin"
      }
    },
    "interaction_options": {
      "help": {
        "prompt": "Help the NPC",
        "result": "You help them. They're grateful.",
        "changes_npc_state": "helped",
        "gives_item": "reward_item"
      },
      "ignore": {
        "prompt": "Ignore the NPC",
        "result": "You walk away.",
        "changes_npc_state": "ignored"
      }
    },
    "possible_fates": ["saved", "died_natural", "died_hazard"],
    "current_fate": null,
    "health": 100,
    "fear_level": 50,
    "can_follow_player": false,
    "inventory": ["item_key"],
    "questgiver": false,
    "quest_id": null,
    "death_message": "How Death claims this NPC if they die",
    "survival_message": "How this NPC escapes if they survive"
  }
}
```

### Field Descriptions

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | ✅ Yes | Display name (e.g., "Corbin Wainwright") |
| `description` | string | ✅ Yes | Physical appearance, demeanor |
| `initial_state` | string | ❌ No | Starting state (conscious, unconscious, injured) |
| `current_state` | string | ❌ No | Runtime state (changes based on events) |
| `location` | string | ❌ No | Room name where NPC is found |
| `is_visionary` | boolean | ❌ No | Whether NPC has death visions |
| `personality_traits` | array | ❌ No | Traits affecting dialogue (cautious, brave, panicked) |
| `dialogue` | object | ✅ Yes | Dialogue tree (see Dialogue Trees section) |
| `interaction_options` | object | ❌ No | Player action choices (help, ignore, question) |
| `possible_fates` | array | ❌ No | Outcomes for this NPC (saved, died_natural, died_hazard) |
| `current_fate` | string | ❌ No | Runtime fate (null until determined) |
| `health` | int | ❌ No | HP (0-100, default: 100) |
| `fear_level` | int | ❌ No | Fear (0-100, affects dialogue tone) |
| `can_follow_player` | boolean | ❌ No | Whether NPC can move rooms with player |
| `inventory` | array | ❌ No | Items NPC is carrying |
| `questgiver` | boolean | ❌ No | Whether NPC gives quests |
| `quest_id` | string | ❌ No | Quest ID if questgiver |
| `death_message` | string | ❌ No | Narrative text if NPC dies |
| `survival_message` | string | ❌ No | Narrative text if NPC survives |

## Dialogue Trees

Dialogue trees define what NPCs say and how conversations branch based on conditions.

### Basic Dialogue

**Single Exchange:**
```json
{
  "corbin_wainwright": {
    "name": "Corbin Wainwright",
    "dialogue": {
      "first_encounter": {
        "text": "He lies motionless in the hospital bed, breathing through a ventilator. His eyes flicker open briefly when you approach."
      },
      "repeat_encounter": {
        "text": "Corbin is still unconscious, the ventilator humming steadily beside him."
      }
    }
  }
}
```

**Behavior:**
```
Player: "talk to corbin"
First time: [first_encounter text]
Subsequent times: [repeat_encounter text]
```

### Branching Dialogue

**Multiple Dialogue Nodes:**
```json
{
  "dr_ellis": {
    "name": "Dr. Ellis",
    "dialogue": {
      "first_encounter": {
        "text": "A doctor in bloodstained scrubs looks up. 'You're still alive? We need to get out of here. Now.'",
        "options": [
          {
            "key": "ask_what_happened",
            "prompt": "What happened here?",
            "response": "I don't know! Patients started dying—one after another. Accidents. Impossible accidents. Then I saw... something. A figure.",
            "next_node": "explain_deaths"
          },
          {
            "key": "ask_escape_route",
            "prompt": "How do we escape?",
            "response": "The main entrance is blocked. We need to find another way. The emergency exit, maybe?",
            "next_node": "discuss_escape"
          },
          {
            "key": "leave",
            "prompt": "I have to go.",
            "response": "Wait! Don't leave me here alone!",
            "next_node": null
          }
        ]
      },
      "explain_deaths": {
        "text": "Every death looked like an accident. A gas leak. A ventilator malfunction. But they're not accidents. There's a pattern.",
        "options": [
          {
            "key": "ask_pattern",
            "prompt": "What pattern?",
            "response": "I can't explain it. But I know we're next if we don't leave. Now.",
            "next_node": "discuss_escape"
          }
        ]
      },
      "discuss_escape": {
        "text": "The emergency exit should be through the ER. But we'll need to navigate past... whatever is causing this.",
        "options": [
          {
            "key": "agree_work_together",
            "prompt": "Let's go together.",
            "response": "Thank you. I won't slow you down, I promise.",
            "changes_npc_state": "following",
            "next_node": null
          },
          {
            "key": "refuse",
            "prompt": "I work alone.",
            "response": "Fine. I hope you make it out.",
            "changes_npc_state": "alone",
            "next_node": null
          }
        ]
      }
    }
  }
}
```

**Behavior:**
```
Player: "talk to dr_ellis"
> [first_encounter text]
> [1] What happened here?
> [2] How do we escape?
> [3] I have to go.

Player: "1"
> [explain_deaths text]
> [1] What pattern?

Player: "1"
> [discuss_escape text]
> [1] Let's go together.
> [2] I work alone.

Player: "1"
> [agree_work_together response]
> [Dr. Ellis will now follow you]
```

### Conditional Dialogue

**Based on Game State:**
```json
{
  "nurse_garcia": {
    "name": "Nurse Garcia",
    "dialogue": {
      "first_encounter": {
        "text": "A nurse huddles in the corner, visibly shaken."
      },
      "if_ventilator_cleared": {
        "text": "You stopped the ventilator? How? That thing was a death trap!",
        "condition": "hazard_cleared:malfunctioning_ventilator"
      },
      "if_mri_cleared": {
        "text": "The MRI is powered down? Thank God. People were getting pulled into that room...",
        "condition": "hazard_cleared:mri_malfunction"
      },
      "if_player_has_journal": {
        "text": "Is that Corbin's journal? He... he knew, didn't he? He saw what was coming.",
        "condition": "player_has_item:corbins_journal"
      },
      "if_high_fear": {
        "text": "I can't take this anymore! I can't! We're all going to die!",
        "condition": "player_fear > 70"
      }
    }
  }
}
```

**Condition Types:**
- `hazard_cleared:<hazard_key>`: Hazard reached terminal success state
- `hazard_active:<hazard_key>`: Hazard currently active
- `player_has_item:<item_key>`: Item in player inventory
- `player_fear > X`: Player fear level threshold
- `player_health < X`: Player health threshold
- `npc_state:<state>`: NPC in specific state
- `quest_completed:<quest_id>`: Quest finished
- `achievement_unlocked:<achievement_id>`: Achievement earned

## NPC States & Fates

NPCs have dynamic states that change based on player actions and hazard outcomes.

### Common NPC States

| State | Description | Dialogue Tone |
|-------|-------------|---------------|
| `conscious` | Aware, can talk normally | Calm or fearful |
| `unconscious` | Cannot interact | No dialogue |
| `injured` | Wounded, limited interaction | Pained, urgent |
| `panicked` | Extremely fearful | Frantic, irrational |
| `helped` | Player assisted them | Grateful, helpful |
| `ignored` | Player abandoned them | Bitter, resigned |
| `following` | Traveling with player | Supportive, nervous |
| `alone` | Refused player help | Independent, wary |
| `dying` | Critical condition | Desperate, final words |
| `dead` | Deceased | No dialogue (examine corpse) |
| `escaped` | Successfully left level | Relief, gratitude |

### State Transitions

**Example: Injured → Helped → Following**
```json
{
  "security_guard_mason": {
    "name": "Security Guard Mason",
    "initial_state": "injured",
    "dialogue": {
      "injured": {
        "text": "A security guard lies on the floor, bleeding from a head wound. 'Help... I can't... stand...'",
        "condition": "npc_state:injured"
      },
      "helped": {
        "text": "Mason struggles to his feet, leaning on you. 'Thanks. I owe you one.'",
        "condition": "npc_state:helped"
      },
      "following": {
        "text": "Mason stays close behind you, one hand on his holstered radio.",
        "condition": "npc_state:following"
      }
    },
    "interaction_options": {
      "help": {
        "prompt": "Help Mason to his feet",
        "result": "You bandage his wound and help him stand. He can walk now.",
        "changes_npc_state": "helped",
        "requires_item": "medical_supplies"
      },
      "leave": {
        "prompt": "Leave him",
        "result": "You leave Mason on the floor. He watches you go, eyes full of fear.",
        "changes_npc_state": "ignored"
      }
    }
  }
}
```

**Flow:**
```
1. Player enters room → Mason is "injured"
2. Player: "help mason" → Requires medical_supplies
3. If player has item: Mason → "helped"
4. Player: "talk to mason" → Dialogue options include "Come with me"
5. Player chooses: Mason → "following"
```

## NPC Interaction Commands

### Talk

**Command:** `talk to <npc>` or `speak to <npc>`

**Behavior:**
```
1. Check if NPC is in current room
2. Check NPC current_state (must be conscious, not dead)
3. Evaluate dialogue conditions
4. Show appropriate dialogue node
5. If dialogue has options, present choices
6. If dialogue gives item, add to inventory
7. If dialogue changes NPC state, update state
```

**Example:**
```
Player: "talk to corbin"
> Corbin stirs slightly, his eyes focusing on you.
> With effort, he reaches out, clutching a worn
> leather journal. He pushes it toward you.
> 
> [Item added: Corbin's Journal]
> [Achievement Unlocked: The Visionary]
```

---

### Help

**Command:** `help <npc>` or `assist <npc>`

**Behavior:**
```
1. Check if NPC has interaction_options["help"]
2. Check if player meets requirements (items, conditions)
3. Apply result (state change, item gift)
4. Show result message
```

**Example:**
```
Player: "help mason"
> You need Medical Supplies to help Mason.

[Player acquires Medical Supplies]

Player: "help mason"
> You bandage Mason's wound and help him stand.
> He can walk now, though he's still weak.
> 
> [Mason's state: injured → helped]
```

---

### Follow

**Command:** `<npc> follow me` or `bring <npc>`

**Behavior (if can_follow_player: true):**
```
1. Check if NPC can_follow_player
2. Check if NPC is in appropriate state (helped, conscious)
3. Set NPC state to "following"
4. NPC moves with player to new rooms
5. NPC appears in room descriptions
```

**Example:**
```
Player: "dr_ellis follow me"
> Dr. Ellis nods. "I'm right behind you."
> [Dr. Ellis is now following you]

Player: "move north"
> You enter the ER Hallway. Dr. Ellis follows close behind.
```

---

### Examine (Dead NPCs)

**Command:** `examine <npc>` when NPC state is "dead"

**Behavior:**
```
1. Check if NPC is dead
2. Show death_message or default death description
3. List items on corpse (can be taken)
```

**Example:**
```
[Ventilator hazard kills Corbin]

Player: "examine corbin"
> Corbin lies motionless on the hospital bed. The
> ventilator is silent. His journal rests on his chest.
> 
> You can take: Corbin's Journal
```

## Survivor Fates

### Fate Determination

**Factors Affecting Fate:**
1. **Player Actions**: Help, ignore, abandon
2. **Hazard Outcomes**: NPC killed by hazard vs. player saves them
3. **Quest Completion**: Did player fulfill NPC's request?
4. **Time Constraints**: Did player reach NPC in time?
5. **Resource Allocation**: Did player use items to save NPC?

### Fate Examples

**Corbin Wainwright (Visionary):**
```json
{
  "corbin_wainwright": {
    "possible_fates": [
      "saved",           // Player powers down ventilator in time
      "died_hazard",     // Ventilator fails, Corbin dies
      "died_natural",    // Corbin's injuries claim him
      "unknown"          // Player never entered room
    ]
  }
}
```

**Fate Logic:**
```python
# If player clears ventilator hazard:
if hazard["malfunctioning_ventilator"]["outcome"] == "success":
    corbin["current_fate"] = "saved"

# If ventilator fails:
elif hazard["malfunctioning_ventilator"]["outcome"] == "failure":
    corbin["current_fate"] = "died_hazard"

# If player never enters room:
elif not room["Patient Room 101"]["visited"]:
    corbin["current_fate"] = "unknown"

# Default:
else:
    corbin["current_fate"] = "died_natural"
```

### Fate Score Impact

**Scoring System:**
```python
fate_scores = {
    "saved": +100,
    "died_natural": -25,
    "died_hazard": -50,
    "sacrificed_self": +25,
    "left_behind": -75,
    "unknown": 0
}

total_npc_score = sum(fate_scores[npc["current_fate"]] for npc in npcs)
```

**Example:**
```
Level 1 NPCs:
- Corbin Wainwright: saved (+100)
- Dr. Ellis: died_hazard (-50)
- Nurse Garcia: left_behind (-75)

Total NPC Score: -25 points
```

### Multiple Ending Paths

**Corbin's Endings:**

**Best Ending: Saved**
```
You managed to power down the ventilator before it
failed. Corbin Wainwright survived, his visions of
death proven wrong—at least this time. He'll carry
the weight of his foreknowledge, but he's alive.

Your intervention broke Death's design.
```

**Tragic Ending: Died to Hazard**
```
The ventilator failed. Corbin Wainwright's lungs
collapsed, just as he'd foreseen. His journal—his
final testament—warned you. But warnings aren't
enough to stop Death's design.

He saw his own death coming. And it came anyway.
```

**Neutral Ending: Unknown**
```
You never found Corbin Wainwright. Perhaps he's
still in that room, waiting. Perhaps Death has
already claimed him. You'll never know.

Sometimes the worst fate is not knowing.
```

## Death's Dialogue

Death is a special NPC—the primary antagonist and narrator.

### Death's Role

**As Narrator:**
- Introduces disasters (opening cinematics)
- Comments on player failures
- Taunts player during hazards
- Provides ominous foreshadowing

**As Character:**
- Appears in visions, reflections, shadows
- Cannot be killed or stopped
- Patient, inevitable, creative

### Death's Dialogue Examples

**Opening (Train Derailment):**
```
"February 19th, 2019. A commuter train derails
outside the city. Metal screams. Glass shatters.
Passengers are thrown like ragdolls.

87 dead. One survivor.

Impossible odds. Impossible luck. Or... perhaps
not luck at all. Perhaps someone cheated.

I don't like cheaters."
```

**During Hazard:**
```
[Player struggling with MRI]
"The magnetic field is beautiful, isn't it? So
orderly. So inevitable. Everything metal will be
drawn to it. Including that scalpel in your pocket.

I wonder how fast it will move."
```

**After Player Success:**
```
[Player defuses bomb]
"Clever. You stopped the bomb. But you can't stop
time. You can't stop entropy. You can't stop me.

All you've done is delay the inevitable. And that
makes it so much sweeter when I finally claim you."
```

**After Player Failure:**
```
[Player dies to gas leak]
"Asphyxiation. A quiet death. No screams. Just...
silence. And then nothing.

You should have smelled the gas. You should have
noticed the signs. But you didn't.

Design perfected."
```

---

### Death's Personality

**Traits:**
- **Patient**: Never hurried, always waiting
- **Creative**: Elaborate Rube Goldberg deaths
- **Fair**: Provides omens, warnings, clues
- **Inevitable**: Cannot be defeated, only delayed
- **Personal**: Remembers every player "victory"

**Tone:**
- Calm, measured, poetic
- Occasionally darkly humorous
- Never angry (frustration is beneath Death)
- Admiring of clever players (worthy opponents)

### Common Issues

#### **NPC Not Responding to Talk Command**
**Symptoms:** `talk to <npc>` → "You don't see anyone by that name."  
**Causes:**
- NPC not in current room (location mismatch)
- NPC key typo (doesn't match npcs.json)
- NPC state is "dead" or "unconscious"

**Debug:**
```python
# Check NPC location:
npc["location"]  # Should match current room

# Check NPC state:
npc["current_state"]  # Should be "conscious" or similar

# Check room NPCs:
room["npcs"]  # Should contain "npc_key"
```

---

#### **Dialogue Conditions Not Working**
**Symptoms:** Expected dialogue doesn't show despite meeting condition  
**Causes:**
- Condition syntax error
- Game state doesn't actually match condition
- Dialogue node not evaluated in order

**Debug:**
```python
# Check condition:
dialogue["condition"]  # Should be valid syntax

# Check game state:
hazard["outcome"]  # For hazard_cleared conditions
player.inventory  # For player_has_item conditions
player.fear  # For player_fear conditions
```

---

#### **NPC Not Giving Item**
**Symptoms:** Dialogue triggers but item not added to inventory  
**Causes:**
- `gives_item` field missing or wrong item key
- Item already in inventory (duplicate prevention)
- Item doesn't exist in items.json

**Debug:**
```python
# Check gives_item:
dialogue["gives_item"]  # Should match items.json key

# Check item definition:
items.json["item_key"]  # Must exist

# Check inventory:
player.inventory  # Should contain item after dialogue
```

---

#### **NPC Fate Not Assigned**
**Symptoms:** NPC fate remains null at end of level  
**Causes:**
- Fate logic not implemented for this NPC
- Hazard outcome doesn't trigger fate assignment
- Player never interacted with NPC

**Debug:**
```python
# Check possible_fates:
npc["possible_fates"]  # Should have valid fate keys

# Check current_fate:
npc["current_fate"]  # Should be assigned by end of level

# Check fate assignment logic:
# Look for code that sets npc["current_fate"] = "..."
```

---

### Testing Checklist

**Before Committing New NPC:**
- [ ] NPC key is unique (no duplicates)
- [ ] `name` is memorable and distinct
- [ ] `description` is evocative (2-3 sentences)
- [ ] `initial_state` is appropriate
- [ ] `location` matches existing room name
- [ ] Dialogue has `first_encounter` and `repeat_encounter`
- [ ] Dialogue conditions are valid and testable
- [ ] `gives_item` items exist in items.json
- [ ] `interaction_options` have clear prompts and results
- [ ] `possible_fates` cover all outcomes
- [ ] `death_message` and `survival_message` are written
- [ ] NPC adds to narrative (not just functional)
- [ ] Personality is consistent across dialogue nodes
- [ ] No typos in NPC key or references

```json
{
  "ICU Waiting Area": {
    "npcs": ["waiting_room_patient"]
  }
}
```

**Usage:**
```
Player: "enter waiting area"
> A small waiting area with uncomfortable plastic chairs.
> An elderly man sits in a wheelchair, staring blankly
> at the wall. He doesn't seem to notice you.

Player: "talk to elderly patient"
> The man doesn't respond. His breathing is shallow.
> You don't think he's aware of anything happening
> around him.

Player: "examine elderly patient"
> He looks very ill. There's nothing you can do for him.
```
