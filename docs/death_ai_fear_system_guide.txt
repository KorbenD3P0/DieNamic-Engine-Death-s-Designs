## Overview

The Death AI & Fear System transforms Death from a passive game-over mechanic into an **active antagonist** that:
- **Narrates** the player's journey
- **Taunts** the player's failures
- **Respects** the player's successes
- **Adapts** to player behavior
- **Punishes** cheating or exploitation

### Design Philosophy

**Death should feel:**
- **Omniscient**: Knows everything happening in the game
- **Personal**: Comments directly on player choices
- **Fair but Ruthless**: Respects skill, punishes mistakes
- **Dynamic**: Adapts dialogue to context (fear level, hazards, NPCs)
- **Inevitable**: Players can delay but not permanently escape

**Death should NOT feel:**
- Random or unpredictable (consistent rules)
- Unfair or cheap (telegraphed threats)
- Repetitive (varied dialogue)
- Silent (always has something to say)

### Core Responsibilities

| System | Responsibility |
|--------|----------------|
| **Death AI** | Dialogue generation, player tracking, cheating detection |
| **Fear System** | Fear accumulation, thresholds, fear-based events |
| **Narrative Generator** | Context-aware Death dialogue, taunts, warnings |
| **Death Sequences** | Game over screens, death animations, respawn logic |

## Death as Entity

### Death's Character Profile

**Identity:**
- Personification of death itself
- Narrator and antagonist
- Architect of "The Design" (fate's plan)

**Personality:**
- **Confident**: Knows most players will fail
- **Amused**: Enjoys watching struggles
- **Respectful**: Acknowledges genuine skill
- **Vengeful**: Hates cheaters and rule-breakers
- **Patient**: Willing to wait for inevitable mistakes

**Voice:**
- Formal but conversational
- Sarcastic when player fails
- Impressed when player succeeds
- Cold and threatening when player cheats

## Death AI Logic

### AI State Machine

**Death AI States:**

```python
class DeathAIState(Enum):
    OBSERVING = "observing"          # Silently watching
    TAUNTING = "taunting"            # Mocking player
    WARNING = "warning"              # Foreshadowing danger
    RESPECTING = "respecting"        # Acknowledging skill
    ANGRY = "angry"                  # Player cheated
    CLAIMING = "claiming"            # Player died
```

**State Transitions:**
```python
# Player performing well → RESPECTING
if hazards_cleared > 3 and npcs_saved > 1 and deaths == 0:
    death_ai_state = DeathAIState.RESPECTING

# Player struggling → TAUNTING
if deaths > 2 or fear > 60:
    death_ai_state = DeathAIState.TAUNTING

# Player near danger → WARNING
if hazard_proximity and fear > 40:
    death_ai_state = DeathAIState.WARNING

# Player cheating → ANGRY
if cheating_detected:
    death_ai_state = DeathAIState.ANGRY

# Player died → CLAIMING
if player_health <= 0:
    death_ai_state = DeathAIState.CLAIMING
```

### Dialogue Probability System

**Death doesn't speak constantly** (becomes annoying). Uses probability:

```python
def should_death_speak(event_type):
    """Determine if Death should comment on event."""
    
    base_probabilities = {
        "hazard_cleared": 0.3,      # 30% chance
        "npc_died": 0.7,            # 70% chance
        "qte_failed": 0.4,          # 40% chance
        "room_entered": 0.1,        # 10% chance
        "player_death": 1.0,        # 100% chance (always)
    }
    
    # Modify by Death AI state
    if death_ai_state == DeathAIState.TAUNTING:
        probability *= 1.5  # More talkative when taunting
    elif death_ai_state == DeathAIState.OBSERVING:
        probability *= 0.5  # Less talkative when observing
    
    # Modify by fear level (higher fear = more Death)
    probability *= (1.0 + player.fear / 100)
    
    return random.random() < probability
```

### Context-Aware Dialogue Selection

**Death chooses dialogue based on:**
- Current Death AI state
- Player fear level
- Recent events (hazard cleared, NPC died, etc.)
- Player stats (deaths, successes, time played)
- Current location (room type, level)

**Example Logic:**
```python
def generate_death_dialogue(event_type, context):
    """Generate appropriate Death dialogue for event."""
    
    # Filter dialogue pool by event type
    dialogue_pool = death_dialogue[event_type]
    
    # Filter by Death AI state
    dialogue_pool = [d for d in dialogue_pool if d["state"] == death_ai_state]
    
    # Filter by fear level
    if player.fear < 40:
        dialogue_pool = [d for d in dialogue_pool if "low_fear" in d["tags"]]
    elif player.fear > 60:
        dialogue_pool = [d for d in dialogue_pool if "high_fear" in d["tags"]]
    
    # Filter by context (if available)
    if context.get("hazard_name"):
        dialogue_pool = [d for d in dialogue_pool 
                         if context["hazard_name"] in d.get("hazards", [])]
    
    # Pick random from filtered pool
    if dialogue_pool:
        return random.choice(dialogue_pool)["text"]
    else:
        return default_death_dialogue[event_type]
```

```python
death_dialogue_database = [
    {
        "text": "Your heart races. I'm getting closer...",
        "event": "fear_threshold",

    },

    },
    {

        "event": "cheating_detected",
        "state": "angry",
        "tags": ["cheating", "punishment"],
    },
]

# Filter example:
def get_dialogue(event, state, fear, hazards_cleared):
    matches = [
        d for d in death_dialogue_database
        if d["event"] == event
        and d["state"] == state
        and d.get("fear_min", 0) <= fear
        and d.get("hazards_cleared_min", 0) <= hazards_cleared
    ]
    return random.choice(matches) if matches else None
```

### Personality Consistency

**Death's voice should remain consistent:**

**DO:**
- Formal language ("you have failed" not "you screwed up")
- Third-person references ("the Design" not "my plan")
- Theatrical ("How delightful!" not "cool")
- Patient tone ("Eventually..." not "hurry up")

**DON'T:**
- Modern slang ("lol", "yeet", "sus")
- Breaking character (apologizing, being friendly)
- Explaining mechanics (immersion-breaking)
- Repetitive phrases (same taunt twice in a row)

### Respawn Logic

**Checkpoint System:**
```python
checkpoints = {
    "level_start": {
        "room": "entrance",
        "health": 100,
        "fear": 20,
        "inventory": [],
        "hazards_cleared": [],
    },
    "mid_level": {

def respawn_player():
    """Respawn player at last checkpoint."""
    
    checkpoint = get_last_checkpoint()
    
    # Restore state
    player.current_room = checkpoint["room"]
    player.health = checkpoint["health"]
    player.fear = checkpoint["fear"]
    player.inventory = checkpoint["inventory"].copy()
    
    # Don't reset hazards (already cleared)
    for hazard_id in checkpoint["hazards_cleared"]:
        hazards[hazard_id]["state"] = "terminal_success"
    
    # Increment death counter
    player.deaths += 1
    
    # Death commentary on respawn
    if player.deaths == 1:
        death_speaks("Back already? Let's see if you do better this time.")
    elif player.deaths == 3:
        death_speaks("Three deaths. You're persistent, I'll give you that.")
    elif player.deaths >= 5:
        death_speaks("How many times must I claim you before you give up?")
```

### Cheat Detection Logic

**Example: Save Scumming Detection**
```python
def detect_save_scumming():
    """Detect if player is excessively reloading saves."""
    
    current_time = time.time()
    
    # Track reload timestamps
    if "reload_timestamps" not in player.metadata:
        player.metadata["reload_timestamps"] = []
    
    # Add current reload
    player.metadata["reload_timestamps"].append(current_time)
    
    # Check last 10 minutes
    recent_reloads = [
        t for t in player.metadata["reload_timestamps"]
        if current_time - t < 600  # 10 minutes
    ]
    
    # If >5 reloads in 10 minutes, trigger punishment
    if len(recent_reloads) > 5 and not player.metadata.get("save_scum_warned"):
        trigger_cheating_punishment("save_scumming")
        player.metadata["save_scum_warned"] = True
```

**Example: Infinite Item Detection**
```python
def on_item_taken(item_id, room_id):
    """Track item taking for duplication exploits."""
    
    # Track item takes
    if "item_takes" not in player.metadata:
        player.metadata["item_takes"] = {}
    
    key = f"{item_id}_{room_id}"
    player.metadata["item_takes"][key] = player.metadata["item_takes"].get(key, 0) + 1
    
    # If same item taken >3 times, player is duplicating
    if player.metadata["item_takes"][key] > 3:
        trigger_cheating_punishment("infinite_items")
        # Remove item from game
        items[item_id]["exists"] = False
```

### Cheating Punishment System

**Escalating Consequences:**

```python
def trigger_cheating_punishment(cheat_type):
    """Punish player for cheating."""
    
    if cheat_type == "save_scumming":
        death_speaks(
            "I see you trying to undo my work. The Design "
            "doesn't allow for do-overs. Let's make this "
            "interesting..."
        )
        # Increase hazard difficulty
        for hazard in hazards.values():
            hazard["qte_time"] *= 0.8  # 20% less time
        player.fear += 30
    
    elif cheat_type == "infinite_items":
        death_speaks(
            "Breaking the laws of reality, are we? I think "
            "NOT. That item is mine now."
        )
        # Remove exploited item
        # Already done in detection
    
    elif cheat_type == "invulnerability":
        death_speaks(
            "YOU DARE TO MAKE YOURSELF INVINCIBLE? I AM DEATH. "
            "I decide who lives and who dies. And you... you DIE."
        )
        # Instant kill, no respawn at checkpoint (restart level)
        player.health = 0
        player.can_respawn = False
        trigger_player_death("cheating_punishment")
    
    elif cheat_type == "room_skipping":
        death_speaks(
            "You think you can skip my carefully crafted Design? "
            "Let me show you what happens when you take shortcuts."
        )
        # Teleport to dangerous hazard room
        teleport_to_room("hazard_gauntlet")
        spawn_hazard("surprise_hazard")
```

## Difficulty Scaling

```python
def calculate_difficulty_modifier():
    """Adjust difficulty based on player performance."""
    
    # Base difficulty
    modifier = 1.0
    
    # Player doing too well
    if deaths == 0 and hazards_cleared > 3 and fear < 30:
        modifier = 1.2  # Increase difficulty by 20%
        death_speaks("You're doing too well. Time to raise the stakes.")
    
    # Player struggling
    elif deaths > 3 and hazards_cleared < 2 and fear > 60:
        modifier = 0.8  # Decrease difficulty by 20%
        death_speaks("Let's give you a fighting chance. I want this to be fun.")
    
    return modifier

# Apply modifier:

fear_gain *= difficulty_modifier
```

### Common Issues

#### **Death Speaks Too Often**
**Symptoms:** Death dialogue appearing constantly, breaking immersion  
**Causes:**
- Probability checks too high
- Death AI state stuck in TAUNTING
- Event triggers not properly rate-limited

**Debug:**
```python
# Check dialogue probabilities:
print(f"Base probability: {base_probabilities[event_type]}")
print(f"Modified probability: {probability}")
print(f"Should speak: {should_death_speak(event_type)}")

# Check Death AI state:
print(f"Death AI State: {death_ai_state}")
print(f"State duration: {time_in_state}")

# Add cooldown:
if time.time() - last_death_dialogue_time < 30:  # 30 second cooldown
    return False
```

#### **Death Never Speaks**
**Symptoms:** No Death dialogue appearing, game feels lifeless  
**Causes:**
- Probability checks too low
- Dialogue pool empty after filtering
- Event triggers not firing

```python
# Check dialogue pool:
print(f"Dialogue pool size: {len(dialogue_pool)}")

# Check event triggers:
```

```python
def on_save_loaded():
    """Detect save scumming and punish if excessive."""
    
    # Track reload
    player.metadata["reload_timestamps"].append(time.time())
    
    # Check for save scumming
    recent_reloads = count_recent_reloads(window=600)  # 10 minutes
    
    if recent_reloads > 5 and not player.metadata.get("save_scum_warned"):
        # Death AI to ANGRY state
        death_ai.state = DeathAIState.ANGRY
        
        # Angry dialogue
        death_dialogue = (
            "I SEE YOU.\n\n"
            "Trying to undo my work. Trying to cheat the Design.\n\n"
            "The Design doesn't allow for do-overs.\n\n"
            "Let's make this INTERESTING."
        )
        display_death_punishment(death_dialogue)
        
        # Punishment: Increase hazard difficulty
        for hazard in hazards.values():
            hazard["qte_time"] *= 0.8  # 20% harder
        
        # Fear penalty
        player.fear += 30
        
        # Mark warned
        player.metadata["save_scum_warned"] = True
        
        # Notification
        show_notification("[All hazards are now more difficult.]")
        show_notification("[Fear +30%]")
        show_notification("[You have been warned.]", color="red")
```
